STS theme pack — documentation

Lives in folder theme wrapper. This describes sts-theme.css (and the generated sts-theme.min.css build for production): portable styles derived from the Slay the Spire Combat Planner reference theme. Pair with theme.html for a live gallery of components.

Installation

Copy this folder’s sts-theme.css into your project assets when reusing elsewhere. Link it from HTML or import from your CSS bundler (plain CSS, no deps). 

<link rel="stylesheet" href="/path/to/your/copy/of/sts-theme.css" />
<!-- or: sts-theme.min.css (smaller transfer, same rules) -->

Recommended baseline for pages using this palette (matches the STS Planner Next.js root layout intent): 

<html lang="en">
<head>
  <link rel="stylesheet" href="sts-theme.css" />
</head>
<body style="min-height:100vh;background:var(--app-bg);color:var(--app-fg);font-family:Inter,system-ui,sans-serif;">
</body>
</html>
Tokens such as --primary use shadcn-style semantics. Combat panels mix fixed RGBA values for the game UI; semantic buttons also use CSS variables so they respond to light/dark when html has class .dark.

Design tokens (:root)

Declared at the root and overridden under .dark (see § Dark tokens). Numeric radii often pair with --radius (0.625rem).

Variable Role
--background Page / surface background in token-driven UIs.
--foreground Default text on --background.
--primary / --primary-foreground Primary solid actions (.btn-primary-solid).
--secondary / --secondary-foreground Secondary solid buttons.
--muted / --muted-foreground Muted surfaces and de-emphasized text.
--accent / --accent-foreground Highlighted surfaces.
--destructive / --destructive-foreground Destructive actions (.btn-destructive).
--border Default hairline borders in token layouts.
--input-background Neutral input fills when using design-token forms.
--ring Focus ring reference (combine with outlines in your app).
--chart-1 … --chart-5 Sequential chart accents (OKLCH).
--sidebar* Sidebar token set (rail layouts).
--font-size, --font-weight-* Baseline typography scale knobs.

App shell variables

In addition to the shadcn-style tokens above, this pack declares app-specific defaults that mirror Tailwind slate usage in the STS Planner layout (bg-slate-950, text-slate-100):

--app-bg #020617 — main shell background.
--app-fg #f1f5f9 — primary foreground on the shell.

Glyphs & icons

Logger & intents use emoji prefixes from ACTIVITY_LOG_ICONS and ENEMY_INTENT_ACTION_GLYPH (app/utils/enemyIntentGlyphs.ts). Timeline / planner UI buttons use lucide-react SVGs — the theme gallery uses .alog-icon-btn-mock only as a size reference.

Dark tokens

Add class dark to <html> (or any wrapping scope) so .dark rules apply. Colour tokens invert for shadcn-style surfaces; combat RGBA stays visually tuned for dark backdrops regardless. 

<html class="dark">

Reuse checklist (other projects)

  • Prefer Inter (or substitute a neo-grotesque) for parity; load weights 400–700 as needed.
  • Set page background via --app-bg / body or Tailwind slate-950-equivalent.
  • When blending with Tailwind/PostCSS: this file sets box-sizing on * plus a full body rule (see callout above). Scope or override if your base layer conflicts.
  • For React: map design tokens to CSS vars from this file rather than duplicating hex in components.
Note: sts-theme.css includes a global body rule: margin: 0, min-height: 100vh, Inter stack, --app-bg / --app-fg. If your product already styles body, import this pack before or after intentionally so cascade matches your design system.
